# make_samplepoint

`make_samplepoint.py` は、WAV ファイルからループ再生用の終端位置を自動探索し、`smpl` チャンクにループ情報を書き込んだ新しい WAV ファイルを作るためのツールです。

音源制作やサンプラー向けに、指定したループ開始点と似た波形を後方から探し、クリックノイズが出にくい位置へ微調整してから書き出します。

## 主な機能

- 指定したサンプル位置をループ開始点として扱う
- 開始点付近の波形と似た場所を正規化相互相関で探索する
- ループ境界の段差と傾きを見て終端位置を微調整する
- 出力 WAV をループ終端付近でクロップする
- WAV の `smpl` チャンクにループ開始・終了情報を書き込む
- ステレオ入力を左、右、またはミックスのモノラルとして処理する
- 通常モードでは 16-bit / 44100 Hz を超える入力を 16-bit / 44100 Hz へ変換してから処理する
- 必要に応じて 22050 Hz / 8-bit へ変換してから処理する

## 必要環境

- Python 3.10 以上
- NumPy
- SoundFile

依存パッケージは次のようにインストールできます。

```powershell
pip install numpy soundfile
```

## 基本的な使い方

```powershell
python make_samplepoint.py 入力.wav --start 52920
```

例:

```powershell
python make_samplepoint.py samples\lead.wav --start 52920 --window 4096 --min-gap 22050
```

出力先を省略した場合は、入力ファイルと同じフォルダに `_lp` を付けた WAV を作ります。たとえば `samples\lead.wav` は `samples\lead_lp.wav` になります。`--22` を付けた場合は `samples\lead_lp_22.wav` になります。

出力先を明示したい場合は、入力ファイルの次に指定します。

```powershell
python make_samplepoint.py samples\lead.wav samples\lead_looped.wav --start 52920
```

処理が成功すると、次の情報が表示されます。

- 入力と出力のサンプル数
- 元のサンプリングレート
- 元のビット深度
- 処理後のサンプリングレートとビット深度
- 使用したチャンネル
- ループ開始位置
- 探索で見つかった仮のループ終端
- 微調整後の最終ループ終端
- ループ長
- 類似度スコア

## オプション

| オプション | 既定値 | 説明 |
| --- | ---: | --- |
| `--start N` | 必須 | ループ開始位置。サンプル単位で指定します。 |
| `--22` | オフ | 読み込み直後に 22050 Hz / 8-bit 相当へ変換してから探索・書き出しします。 |
| `--window N` | `4096` | 類似度判定に使う波形窓の長さ。大きいほど安定しやすく、小さいほど細かく反応します。 |
| `--min-gap N` | 未指定 | ループ開始位置から最低どれだけ離れた場所から終端探索を始めるか。未指定時は、比較窓の直後から波形末尾まで走査するかを `Y/n` で確認します。 |
| `--search-end N` | なし | 探索範囲の終端。省略時はファイル末尾付近まで探します。 |
| `--threshold F` | `0.85` | 採用する最低類似度。見つからない場合は少し下げると通りやすくなります。 |
| `--refine N` | `1024` | 仮の終端位置の前後何サンプルを微調整範囲にするか。 |
| `--crop-margin N` | `0` | ループ終端後に残すサンプル数。余韻や検証用の後ろ部分を残したい場合に使います。 |
| `--channel MODE` | `left` | ステレオ入力時の扱い。`left`、`right`、`mix` から選びます。 |

## 調整の目安

ループが短すぎる場合は、`--min-gap` を大きくしてください。

`--min-gap` を省略した場合は、比較窓の直後から波形末尾まで走査するかを確認します。

```text
--min-gap was not specified. Scan loop-end candidates from sample 12702 to 17609? [Y/n]:
```

Enter だけ、または `Y` を入力すると探索を続行します。`n` を入力すると処理を中止します。

短い素材で `--min-gap` を明示する場合は、小さめの値を指定してください。たとえば 0.5 秒未満の素材では、次のように実行できます。

```powershell
python make_samplepoint.py samples\lead.wav samples\lead_loop.wav --start 11678 --window 1024 --min-gap 1000
```

22050 Hz / 8-bit に変換してから処理したい場合は、`--22` を付けます。

```powershell
python make_samplepoint.py samples\lead.wav --22 --start 5839 --window 512 --min-gap 500
```

`--22` 使用時は、読み込み直後にサンプリングレートと量子化段数を変換します。そのため `--start`、`--window`、`--min-gap`、`--search-end` は、変換後の 22050 Hz のサンプル単位で指定してください。

通常モードでも、入力が 24-bit / 48000 Hz など 16-bit / 44100 Hz を上回る場合は、読み込み直後に 16-bit / 44100 Hz へ変換してから処理します。この場合の `--start`、`--window`、`--min-gap`、`--search-end` は、変換後の 44100 Hz のサンプル単位で指定してください。

よいループ位置が見つからない場合は、まず `--start` を音量や音色が安定している場所へ動かします。それでも失敗する場合は、`--threshold` を少し下げる、`--window` を大きくする、または `--search-end` で探索範囲を絞ると改善することがあります。

ループ境界でクリックが出る場合は、`--refine` を大きくして微調整範囲を広げてください。素材によっては、`--start` をゼロクロス付近や周期が安定した位置へ移す方が効果的です。

## 出力ファイルについて

出力はモノラルの PCM WAV です。通常は最大 44100 Hz / 16-bit、`--22` 使用時は 22050 Hz / 8-bit で書き出します。ループ情報は WAV の `smpl` チャンクに書き込まれます。

`smpl` チャンクを読むサンプラーや波形編集ソフトでは、書き込まれたループ開始・終了位置を利用できます。ソフトによっては `smpl` チャンクを無視する場合があるため、その場合は対応しているツールで確認してください。

## 注意点

- 入力は WAV ファイルを想定しています。
- `--start`、`--window`、`--min-gap`、`--search-end` はすべてサンプル単位です。
- ループ開始点の波形が無音や平坦に近い場合、類似度を計算できず失敗します。
- ステレオ入力でも出力はモノラルになります。
